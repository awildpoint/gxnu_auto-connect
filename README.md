# 校园网自动认证工具

## 🎯 项目简介

这是一个专为广西师范大学（GXNU）或其他使用特定认证接口的校园网（DrCOM）设计的图形化自动认证工具。该工具能够自动检测网络连接状态，并在断线时尝试自动重连，无需用户手动登录。它提供了直观的图形用户界面（GUI），支持多运营商选择，并可最小化到系统托盘后台运行。

## ✨ 主要功能

- ✅ **自动监控**：后台线程每5分钟检测一次网络连接状态。
    
- ✅ **断线重连**：检测到网络离线时，自动调用校园网登录接口进行重连。
    
- ✅ **图形化界面**：使用Tkinter构建，包含状态监控、配置管理、运行日志三个标签页。
    
- ✅ **配置管理**：支持账号、密码、运营商、IP地址的保存与修改。
    
- ✅ **运行日志**：界面实时显示及文件记录运行状态和事件。
    
- ✅ **系统托盘**：支持最小化到系统托盘，后台静默运行。
    
- ✅ **手动控制**：提供“立即检测”和“立即登录”的按钮。
    
- ✅ **可执行文件构建**：提供 `build.py`脚本，可将项目打包为单文件可执行程序。

## ⚙️ 环境要求与依赖

要运行此项目的源代码，您需要安装以下Python库。具体版本要求如下：

**requirements.txt**

```
pystray>=0.19.0
Pillow>=9.0.0
tkinter  # 通常为Python标准库，无需额外安装
```

> **注**：`tkinter`是Python的标准GUI库，在大多数Python安装中已包含。如果缺失，在Linux上可能需要单独安装（例如：`sudo apt-get install python3-tk`）。

**主要依赖库说明**：

    
- **`pystray`**：用于创建和控制系统托盘图标。
    
- **`Pillow`(PIL)**：用于处理托盘图标等图像文件。
    

您可以通过以下命令安装依赖：

```
pip install requests pystray pillow
```


### 1. 运行源代码

确保已安装上述依赖后，直接运行主程序：

```
python main4.py
```

首次运行会自动在程序同级目录下生成 `config.json`（配置文件）和 `log.txt`（日志文件）。

### 2. 配置程序

1. 程序启动后，点击 **“配置管理”**​ 标签页。
    
2. 填写您的校园网**账号**、**密码**、选择**运营商**，并输入您的本地IPv4地址（**v4ip**）。
    
3. 点击 **“保存配置”**。
    

> **如何获取IP地址**：在命令行输入 `ipconfig`(Windows) 或 `ifconfig`(Linux/Mac)，查找本地连接的IPv4地址。

### 3. 使用程序

- **状态监控**：查看当前账号、运营商、IP、网络在线状态及最后检测时间。
    
- **手动控制**：使用“立即检测网络”和“立即登录”按钮进行手动操作。
    
- **最小化**：点击“最小化到托盘”，程序将在后台继续运行。
    
- **退出**：点击“退出程序”，或从托盘菜单退出。
    

## ⚙️ 配置文件详解

程序通过 `config.json`管理设置，结构如下：

```
{
  "account": "你的学号",
  "password": "你的密码",
  "operator": 0,
  "v4ip": "你的工位ip",
  "login_url": "https://yc.gxnu.edu.cn/drcom/login"
}
```

- **`operator`**: 运营商编号，0=校园网, 1=移动, 2=联通, 3=电信。与界面下拉菜单对应。
    
- **`v4ip`**: **必须填写**。这是校园网认证所需的客户端IP参数。
    
- **`login_url`**: 校园网认证接口地址，默认为广西师范大学的地址。如需用于其他学校，请修改此项。
    

## 🛠️ 构建可执行文件

项目提供了 `build.py`脚本，用于将程序打包成独立的、无需Python环境的可执行文件（.exe）。

### 构建步骤

1. 确保已安装 **PyInstaller**：`pip install pyinstaller`
    
2. 在项目根目录下运行构建脚本：
    
    ```
    python build.py
    ```
    
3. 构建完成后，可在 `dist/`目录下找到 `校园网自动认证工具.exe`。
    

### 构建参数说明 (`build.py`)

- `--onefile`: 打包为单个可执行文件。
    
- `--windowed`: 运行时隐藏控制台窗口（适用于GUI程序）。
    
- `--noconsole`: 同上，不创建控制台窗口。
    
- `--icon=gxnu.ico`: 设置应用程序图标（需存在 `gxnu.ico`文件）。
    
- `--add-data`: 将配置和日志文件打包进可执行文件，运行时会在相同目录释放。
    

## 🔧 高级使用与故障排除

### 自定义认证接口

如果您需要将此工具用于其他学校：

1. 使用浏览器开发者工具（F12）分析您所在学校的登录过程。
    
2. 获取登录请求的URL和参数格式。
    
3. 修改 `main4.py`中 `login()`方法内的 `login_url`和参数拼接逻辑。
    

### 常见问题

1. **登录失败**
    
    - 确认 `config.json`中的 `account`、`password`、`v4ip`填写正确。
        
    - 确认 `operator`选择正确。
        
    - 检查网络是否可访问认证服务器 (`yc.gxnu.edu.cn`)。
        
    
2. **无法检测网络状态**
    
    - 程序通过 `ping www.baidu.com`检测网络。如果百度过于严格，可考虑修改 `check_online()`方法中的检测地址。
        
    
3. **托盘图标不显示**
    
    - 确认已安装 `pystray`和 `Pillow`。
        
    - 某些Windows系统主题或安全软件可能会阻止托盘图标显示。
        
    
4. **打包后无法运行**
    
    - 确保打包时 `config.json`和 `gxnu.ico`存在于源目录。
        
    - 首次运行打包版程序时，请**以管理员身份运行**，以确保能在程序目录创建配置和日志文件。
        
    

### 扩展建议

- **多账号支持**：可修改代码，在 `config.json`中支持账号列表，并在GUI中增加切换功能。
    
- **开机自启**：将打包后的 `.exe`文件快捷方式放入系统启动文件夹。
    
- **网络检测优化**：可增加多个检测地址（如 `114.114.114.114`、`www.qq.com`），提高检测可靠性。
